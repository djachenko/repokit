# decisions — Архитектурные решения проекта init_python

### Один режим запуска — из текущей папки

Отказались от трёх режимов (аргумент / интерактив / из папки) в пользу одного: скрипт всегда работает в текущей директории, имя берёт из `$(basename "$PWD")`. Причина: меньше ветвлений, поведение предсказуемое.

### OWNER из gh api, не хардкод

Было `OWNER="djachenko"` в скрипте. Заменили на `gh api user --jq '.login'` — работает для любого аккаунта, не надо менять при смене машины или форке.

### Шаблоны как отдельные файлы, не heredoc

Workflow-файлы и pyproject.toml живут в `templates/` как настоящие YAML/TOML. Подстановка через `sed` при копировании. Плюсы: подсветка синтаксиса, редактируются независимо, нет проблем с экранированием в heredoc.

### Матрица только в integration, не в tests

`tests.yml` (запускается на каждый push) — без матрицы, быстро. Матрица — только в `integration.yml` (запускается на PR перед мержом). Причина: матрица нужна один раз перед мержом, не на каждый коммит.

### pytest || [ $? -eq 5 ]

Проекты без юнитов не должны ломать CI. Pytest возвращает exit code 5 если тесты не найдены — обрабатываем явно.

### bypass_actors: хардкод App ID автора

`07_ruleset.sh` содержит `actor_id: 2991967` — ID GitHub App автора. Это позволяет release-workflow пушить тег в master в обход ruleset. Для чужих пользователей ID будет другим — нужно либо вынести в `.repokit`, либо резолвить через `gh api /apps/{slug}`. До тех пор инструмент рассчитан на личное использование.

### required_status_checks: один контекст "integration"

Матрица в `integration.yml` создаёт много контекстов. Required check — только финальный job `integration` (depends on `test`), не отдельные матричные джобы.

### Ruleset: GET → DELETE → POST

POST создаёт дубли если ruleset уже есть. Решение: GET по имени, если нашли — DELETE, затем POST. PATCH на GitHub Rulesets API официально не задокументирован и ведёт себя непредсказуемо, поэтому DELETE + POST. Недостаток: короткое окно между DELETE и POST без защиты ветки.

### Ruleset name: не "master"

Имя "master" путается с именем ветки. Менять на что-то описательное (обсуждается).

### Конфликты шаблонов: пропускать по умолчанию

Если workflows/ или pyproject.toml уже существуют — пропускаем, не перетираем. Флаг --force отложен до реальной потребности.
